# 三、准备国产模型 API

本文以 DeepSeek 为例。

其他国产模型流程类似：

- Qwen
- GLM
- Kimi

---

## 第一步：创建 API Key

进入 DeepSeek 开放平台。

注册账号。

进入 API Key 管理页面。

创建新的 API Key。

保存：

```text
你的 API Key
```

注意：

API Key 相当于账户密码。

不要：

- 上传到 GitHub
- 分享给别人
- 写入公开代码

---

# 四、Codex++ 配置 DeepSeek

## 第一步：打开 Codex++

启动 Codex++。

进入：

```text
模型管理
```

或者：

```text
Provider 设置
```

---

## 第二步：添加新的模型 Provider

选择：

```text
OpenAI Compatible
```

因为大部分国产模型都提供兼容 OpenAI API 的接口。

---

填写：

### API Base

```text
https://api.deepseek.com/v1
```

---

### API Key

填写刚才创建的：

```text
DeepSeek API Key
```

---

### Model

填写：

```text
deepseek-chat
```

---

保存配置。

---

# 五、启动并测试

选择刚刚添加的模型。

启动 Codex。

测试：

```text
帮我分析当前项目结构，并指出可以优化的地方。
```

或者：

```text
帮我重构这个函数，提高可读性。
```

如果 Codex 能正常读取项目并返回结果。

说明配置成功。

# 六、如何接入其他国产模型？

配置逻辑基本一样。

核心只有三个参数：

1. API 地址

2. API Key

3. Model 名称

---

# Qwen（通义千问）

配置：

API Base：

```text
根据阿里云百炼 OpenAI 兼容接口地址配置
```

模型：

例如：

```text
qwen-plus
```

或者：

```text
qwen-max
```

适合：

- 中文项目
- 企业应用
- 长文本任务

---

# GLM（智谱）

配置：

API Base：

```text
根据智谱开放平台接口地址配置
```

模型：

例如：

```text
glm-4
```

适合：

- 综合开发任务
- 中文理解
- 推理分析

---

# Kimi

配置：

API Base：

```text
根据 Moonshot 开放平台接口地址配置
```

模型：

选择对应 Kimi 模型。

适合：

- 长代码阅读
- 大型项目分析

# 八、常见问题

## 1. API Key 错误

现象：

```text
401 Unauthorized
```

检查：

- Key 是否复制完整
- 是否过期
- 账户是否有余额

---

## 2. 模型不存在

现象：

```text
Model Not Found
```

原因：

模型名称填写错误。

解决：

查看对应平台最新模型列表。

---

## 3. 可以聊天，但是无法完成 Agent 任务

原因：

不同模型支持能力不同。

例如：

- Tool Calling
- 文件操作
- 多步骤任务

并不是所有模型都支持。

选择支持 Agent 能力的模型。
